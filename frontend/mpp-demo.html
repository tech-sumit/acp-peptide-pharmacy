402 challenge flow

Aligned with mpp.dev/protocol and Stripe SPT charges via Stripe MPP.

  1. Client POST /mcp without payment → 402 + WWW-Authenticate: Payment.
  2. Client completes Stripe test mode payment and builds the credential.
  3. Client retries with Authorization: Payment … → MCP runs; response may include Payment-Receipt.
